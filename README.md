这个仓库是从爬虫项目里沉淀下来的通用工具模块。写爬虫的时候发现有些功能反反复复用到，比如发请求、存文件、打日志这些，每个项目都要写一遍挺烦的，就抽出来单独放一个仓库，需要的时候直接拿过来用。

仓库里都有什么模块

请求.py 负责网络请求这块。Requests和Selenium都包了一层，支持重试、超时设置、自定义请求头。用了requests的Session来保持会话，爬需要登录的网站会方便一些。Selenium那部分支持无头模式和懒加载设置，跑大量数据的时候能省点资源。每个请求函数都有重试逻辑，第一次超时了会自动等几秒再试，避免网络波动导致整个爬虫崩掉。

保存.py 负责把数据存下来。txt、csv、excel、json都支持，给什么数据用什么格式存。最常用的是存txt和csv。txt适合保存文字详情，csv适合保存表格数据。这个模块里还有一个比较实用的功能是自动处理文件名重复，如果文件夹里已经有了同名文件，会自动在后面加序号，不会覆盖掉之前的文件。另外还会自动检查磁盘空间和写入权限，如果盘满了或者没有写权限会提前报错，不至于写到一半才崩。

反爬.py 主要用来降低被封的风险。核心功能是两个，一个是模拟人类输入，打字的时候有快有慢，中文和英文的间隔也不一样，不会像粘贴那样瞬间填完。另一个是随机等待，每次操作之间歇的时间不固定，有长有短，避免被网站识别出固定频率的访问模式。这两个功能单独用效果一般，但配合user-agent轮换和代理使用的话能明显提高存活率。

日志.py 负责打日志。支持把日志同时输出到控制台和文件，方便调试也方便事后查问题。日志按级别分，info记录正常流程，warning记录可恢复的错误，error记录需要关注的问题。还支持单独开一个错误日志文件，专门记录warning以上的日志，排查问题的时候不用在海量info里翻，直接看错误日志就行。

数据.py 目前只做了一个基础的数据清洗函数，主要是去掉字符串里多余的空格、换行、制表符这些。爬下来的数据经常带着一大堆空白符，不处理的话存进文件很难看，搜索也不方便。这个清洗函数会把连续的空白合并成一个空格，头尾的空白直接去掉。如果传进来的是列表，还会递归处理列表里的每个元素。

自动化.py 是辅助浏览器操作的工具。目前主要实现了三个功能，一个是切换浏览器窗口句柄，点开新窗口后自动切到最新的那个，不用自己手动找。一个是模拟人类输入，和反爬模块里那个类似，但这个是作为一个独立的普通函数提供的，不依赖WebElement补丁。还有一个是等待页面加载，定位到某个元素就认为页面加载完成了，比固定sleep智能一些，能节省不少时间。

怎么使用这些工具

最简单的用法是把整个tools文件夹复制到你的项目目录下，然后直接在代码里import。比如 from 请求 import gain_html 就能用里面的请求函数， from 保存 import save_file 就能用保存函数。每个模块的核心函数都有注释，函数名也基本能看出来是干什么的，不太需要翻文档。

如果只想用某一个功能，也可以单独复制那个模块文件，因为这个工具库的设计原则就是模块之间尽量少耦合。比如只想用日志功能，单独复制日志.py就行，不需要把整个tools都搬过来。

依赖哪些东西

Python 3.8 或更高版本。核心依赖是 requests、selenium、pandas、lxml 这几个。完整列表在 requirements.txt 里，用 pip install -r requirements.txt 一键安装。webdriver-manager 会自动管理ChromeDriver的版本，不用自己去下载和配置环境变量，省了不少事。

这些工具的现状

这些工具是我边学边写的，不是一开始就设计好的，所以不同模块的风格和成熟度不太一样。请求模块和保存模块用得最多，迭代次数也最多，相对稳定。反爬和自动化模块用得少一些，功能上够用但覆盖面还不够全，有些场景可能还覆盖不到。后续用着用着发现问题会慢慢改进，但不会追求大而全，够用就行。

如果发现哪个功能不好用或者缺了什么，欢迎提出来，我有空会看看怎么改进。
